# AI Agent 新手课：一个人如何搭起自己的 Builder 工作流 - 执行方案

## 1. 活动时间线

### T-7 天：确定活动框架

- 确定活动主题、活动目标、目标听众和核心讨论问题。
- 确认活动形式：Twitter / X Space、飞书群直播讨论、Discord 语音或其他平台。
- 确定活动时长：建议 60 分钟，最长不超过 75 分钟。
- 确定主持人、主分享人和协助嘉宾。
- 输出第一版活动策划案和活动当天内容流程。

### T-5 天：完成嘉宾与内容准备

- 确认嘉宾是否参与连麦或圆桌讨论。
- 给主持人和嘉宾同步活动定位：面向新人、重实践、少讲抽象概念。
- 准备主持人提纲、开场话术、核心讨论问题和互动环节说明。
- 准备 7 天 Builder 任务模板，作为活动结束后的行动引导。

### T-3 天：启动第一轮宣传

- 发布第一条活动预告。
- 宣传重点：新人为什么需要 AI Agent 工作流、活动能解决什么问题。
- 在社群、朋友圈、Twitter / X、Telegram / Discord 等渠道同步。
- 收集听众问题：让大家提前提交“最想用 AI Agent 解决的问题”。

### T-1 天：活动提醒与内容确认

- 发布第二轮提醒。
- 确认 Space 链接、活动时间、主持人和嘉宾设备状态。
- 主持人完成一次 10-15 分钟走稿。
- 整理提前收集的问题，挑选 3-5 个适合现场讨论的问题。
- 准备活动当天备用文案：开场、转场、互动、CTA、异常情况说明。

### T-0：活动当天

- 活动开始前 30 分钟：主持人、嘉宾和运营人员进入准备状态。
- 活动开始前 15 分钟：确认网络、麦克风、Space 链接、宣传渠道。
- 活动开始前 5 分钟：在社群发送最后提醒。
- 活动进行中：按议程推进，记录高质量观点和听众问题。
- 活动结束后 10 分钟内：发布感谢与 CTA。

### T+1 天：复盘与二次传播

- 整理活动纪要：核心观点、讨论问题、听众反馈、下一步行动。
- 发布活动总结或长推。
- 收集参与者的 7 天 Builder 任务选择。
- 记录活动数据：参与人数、峰值在线、互动次数、问题数量、后续报名/打卡人数。

### T+7 天：成果回收

- 收集听众的 7 天任务成果。
- 挑选 3-5 个优秀案例进行展示。
- 判断是否继续举办成果展示 Space 或后续共学活动。

## 2. 宣传节奏

### 第一轮：主题预告

时间：T-3 天

目标：让目标听众知道活动主题，并产生“这和我有关”的感觉。

宣传重点：

- AI Agent 不只是高级工具，而是新人 Builder 的起步杠杆。
- 活动适合不会代码、刚接触 Web3 / AI、想做作品但不知道如何开始的人。
- 听完不是只获得概念，而是带走一个 7 天可执行任务。

示例文案方向：

> 想做 Builder，但不知道从哪里开始？  
> 这场 Space 我们聊聊：新人如何用 AI Agent 搭起自己的工作流，从想法、研究、原型到发布，完成第一次可展示的小作品。

### 第二轮：问题引导

时间：T-2 天或 T-1 天上午

目标：通过问题收集增加参与感。

宣传重点：

- 邀请听众提前提交问题。
- 强调现场会围绕新人真实卡点讨论。

可收集的问题：

- 你现在最想用 AI Agent 帮你解决什么问题？
- 你做 Builder 最大的阻碍是什么？
- 你更想做技术型、内容型、研究型，还是运营型 Builder？

### 第三轮：开播提醒

时间：T-0，当天活动前 2 小时和活动前 10 分钟

目标：提高到场率。

宣传重点：

- 活动开始时间。
- Space 链接。
- 本场会发布 7 天 Builder 任务。
- 欢迎新人带着问题来听。

### 活动后传播

时间：T+1 天

目标：把一次 Space 转化为可沉淀内容和后续行动。

传播内容：

- 活动纪要。
- 3-5 条高价值观点。
- 7 天 Builder 任务说明。
- 后续打卡或成果展示入口。

## 3. 团队分工

| 角色 | 负责人 | 主要职责 |
|---|---|---|
| 活动负责人 | 待定 | 把控整体目标、时间线、人员协调和最终交付。 |
| 主持人 | 待定 | 控制活动节奏、串联议程、引导讨论、处理冷场。 |
| 主分享人 | 待定 | 讲解 AI Agent 与 Builder 工作流的核心内容。 |
| 嘉宾 / 连麦嘉宾 | 待定 | 分享个人经验，补充不同身份视角，如技术、研究、运营。 |
| 宣传运营 | 待定 | 发布预告、提醒开播、同步社群信息、活动后总结传播。 |
| 互动记录 | 待定 | 收集听众问题、记录高质量观点、整理活动纪要。 |
| 技术支持 | 待定 | 检查 Space 链接、设备、录音、备用沟通渠道。 |

### 分工原则

- 主持人只负责节奏和场面，不承担全部内容输出。
- 主分享人负责把复杂概念讲简单，避免过度技术化。
- 互动记录要实时抓取听众问题，帮助主持人判断哪些问题值得现场回应。
- 宣传运营要提前准备好开播提醒和活动后 CTA，避免活动结束后断掉。

## 4. 活动当天执行流程

### 活动前 30 分钟

- 主持人、主分享人、嘉宾进入准备状态。
- 确认活动链接可正常打开。
- 检查麦克风、耳机、网络和备用设备。
- 宣传运营准备最后一轮提醒文案。
- 互动记录打开记录文档，准备记录问题和观点。

### 活动前 10 分钟

- 在社群和社交平台发布开播提醒。
- 主持人确认嘉宾是否已进入 Space。
- 技术支持确认录音或回放设置。
- 主持人快速复习议程和转场话术。

### 0-5 分钟：开场

- 主持人欢迎听众。
- 介绍活动主题和适合人群。
- 明确本场活动不是工具清单分享，而是帮助新人形成 Builder 工作流。
- 抛出第一个互动问题：你现在最想用 AI Agent 帮你解决什么问题？

### 5-12 分钟：概念澄清

- 主分享人解释 AI Agent 的基本概念。
- 用新人能懂的语言说明它和普通 AI 对话工具的区别。
- 给出 2-3 个 Builder 场景例子：项目研究、任务拆解、原型生成。

### 12-25 分钟：工作流拆解

- 拆解个人 Builder 的完整路径：
  - 发现问题
  - 整理资料
  - 明确目标用户
  - 拆解 MVP
  - 做出最小交付物
  - 发布与复盘
- 强调新人不需要一开始做复杂项目，先完成一次闭环。

### 25-38 分钟：场景示范

- 选择一个示例任务进行拆解，如“做一个 Web3 项目研究卡片”。
- 展示 AI Agent 可以如何参与：
  - 帮忙提出研究问题
  - 整理资料结构
  - 生成初版内容
  - 找出信息缺口
  - 形成发布版本
- 说明技术型新人也可以替换成 DApp 原型或合约 demo。

### 38-48 分钟：核心讨论

- 围绕提前准备的核心问题展开讨论。
- 主持人优先选择新人共性问题，不展开过深技术分支。
- 嘉宾从各自角度补充：技术、内容、研究、运营分别如何起步。

### 48-55 分钟：互动环节

- 开启“7 天 Builder 任务现场认领”。
- 邀请听众选择一个身份标签。
- 主持人根据身份给出任务建议。
- 选择 2-3 位听众的问题进行现场任务缩小。

### 55-60 分钟：总结与 CTA

- 总结本场活动的核心观点：
  - AI Agent 的价值在于帮助新人减少卡住时间。
  - Builder 的第一步不是做大项目，而是完成小闭环。
  - 7 天任务是进入行动状态的最好方式。
- 发布活动后 CTA。
- 引导听众提交自己的 7 天任务选择或加入后续打卡。

### 活动后 10 分钟

- 在社群发布感谢信息和 CTA。
- 同步 7 天任务模板。
- 邀请参与者留言自己的任务方向。
- 记录活动表现和待复盘问题。

## 5. 可能出现的问题与应对方式

### 问题一：新人听不懂 AI Agent 的概念

应对方式：

- 避免一开始讲框架、模型、工具链。
- 用生活化类比：普通 AI 像问答助手，AI Agent 更像能帮你拆任务、记上下文、推进目标的工作搭档。
- 每讲一个概念都配一个 Builder 场景。

### 问题二：讨论太技术化，非技术新人参与感下降

应对方式：

- 主持人及时把问题拉回“新人怎么开始”。
- 准备非技术路径案例：研究卡片、内容复盘、社群活动、用户访谈。
- 强调 Builder 不等于程序员，持续产出才是核心。

### 问题三：听众互动少，现场冷场

应对方式：

- 活动前提前收集问题，准备 5-8 个备用问题。
- 主持人使用选择题式提问，降低参与门槛。
- 先让听众在评论区选择身份标签，再邀请连麦。
- 嘉宾提前准备 1-2 个个人失败或卡点故事，避免分享过于完美。

### 问题四：嘉宾或主持人网络不稳定

应对方式：

- 提前确认备用主持人或备用嘉宾。
- 活动前 30 分钟完成设备检查。
- 准备主持人独立可讲的完整提纲，即使嘉宾掉线也能继续。
- 在社群准备备用沟通渠道。

### 问题五：活动节奏拖延，无法完成 CTA

应对方式：

- 主持人提前设定每个环节的时间上限。
- 到 48 分钟时必须进入互动环节。
- 到 55 分钟时必须进入总结和 CTA。
- 如果讨论很热，可以把未讲完的问题放到后续复盘或下一场活动。

### 问题六：活动结束后没有后续行动

应对方式：

- 活动结束前明确发布 7 天任务。
- 活动后 10 分钟内在社群同步任务模板。
- T+1 天发布活动纪要和任务提醒。
- T+7 天设置成果回收或展示机制。

### 问题七：听众选的任务过大，7 天内做不完

应对方式：

- 主持人现场帮助缩小范围。
- 用“最小交付物”替代“完整项目”。
- 引导听众只保留一个核心目标、一个目标用户、一个可展示结果。
- 示例：不要做“AI Agent 平台”，改成“整理 5 个 AI Agent 工具并写一份新人选择指南”。
