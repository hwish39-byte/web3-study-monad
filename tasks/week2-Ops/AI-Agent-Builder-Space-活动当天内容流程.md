# AI Agent 新手课：一个人如何搭起自己的 Builder 工作流

## 1. 活动标题

**AI Agent 新手课：一个人如何搭起自己的 Builder 工作流**

副标题：从想法、研究、原型到发布，新人如何用 AI Agent 完成第一次 Builder 实践。

## 2. 活动简介

这是一场面向 Web3、AI 和 Builder 新人的 Space 内容分享与讨论。

活动重点不是讲复杂技术概念，也不是追热点名词，而是帮助新人理解：AI Agent 到底能在个人学习、项目研究、产品原型、内容输出和持续迭代中扮演什么角色。

通过这场活动，参与者将看到一个更具体的 Builder 工作流：如何从一个模糊想法开始，用 AI 辅助拆解任务、整理资料、形成方案、推进小作品，并在活动结束后选择一个 7 天内可以完成的实践任务。

## 4. 核心讨论问题

### 问题一：AI Agent 和普通 AI 聊天工具的真正区别是什么？

讨论重点：

- 普通 AI 更像即时问答工具，AI Agent 更强调目标、上下文、任务拆解和连续执行。
- 对新人来说，不需要一开始追求复杂 Agent 系统，先学会把 AI 当作研究助手、产品助理、代码助理和复盘伙伴。
- 判断一个 AI 工作流是否有效，不是看工具多高级，而是看它是否帮你减少卡住的时间。

### 问题二：新人做 Builder，最容易卡在哪一步？

讨论重点：

- 卡在不知道选什么题：可以从自己的学习痛点、社区需求、重复劳动开始。
- 卡在不会拆任务：可以让 AI 帮忙拆成 1 天、3 天、7 天版本。
- 卡在不会写代码：可以先做无代码原型、需求文档、页面草图、研究报告，再逐步进入技术实现。
- 卡在做不完：新人第一阶段应该追求“小而完整”，而不是“大而完美”。

### 问题三：AI Agent 最适合辅助 Builder 的哪些环节？

讨论重点：

- 研究：快速整理项目背景、竞品、术语、用户问题。
- 产品：把模糊想法拆成用户场景、功能列表和 MVP 范围。
- 技术：解释代码、生成脚手架、辅助 debug、写测试用例。
- 内容：生成活动总结、项目介绍、社群公告、推文草稿。
- 复盘：整理做过什么、卡在哪里、下一步怎么迭代。

### 问题四：非技术新人如何参与 AI / Web3 Builder 生态？

讨论重点：

- 不会代码也可以从研究、内容、运营、社区、BD、测试、用户反馈等角色切入。
- AI 可以帮助非技术新人补齐表达、分析、结构化输出和基础原型能力。
- Builder 不等于程序员，Builder 的核心是发现问题、组织资源、持续产出。

### 问题五：如何设计一个 7 天内能完成的 Builder 小任务？

讨论重点：

- 任务必须足够小，有明确交付物。
- 交付物可以是项目研究卡片、页面原型、智能合约 demo、数据看板、工具清单、复盘文章。
- 目标不是证明自己很强，而是完成一次从想法到产出的闭环。

## 5. 互动环节设计

### 环节名称：7 天 Builder 任务现场认领

**互动目标：** 帮助听众从“听懂了”走向“我知道自己接下来做什么”。

**环节流程：**

1. 主持人抛出问题：如果接下来 7 天只能完成一个小作品，你会选择做什么？
2. 听众在评论区或连麦中选择一个身份标签：
   - 技术型 Builder
   - 内容型 Builder
   - 研究型 Builder
   - 运营/社区型 Builder
   - 还没想清楚的新手
3. 主持人根据不同身份给出对应任务建议：
   - 技术型：做一个极简 DApp 页面或合约 demo。
   - 内容型：写一篇 AI Agent 工具实践复盘。
   - 研究型：整理一个 Web3 项目的研究卡片。
   - 运营/社区型：设计一个新人共学打卡活动方案。
   - 还没想清楚的新手：用 AI 生成 3 个可执行的小项目选题，再选一个最小版本。
4. 邀请 2-3 位听众分享自己的选择，主持人现场帮他们缩小任务范围。
5. 最后统一给出一个简单模板：任务名称、目标用户、最终交付物、第一步行动。

**主持人引导语：**

“今天不要求大家立刻成为很成熟的 Builder。我们只做一件事：选一个足够小、7 天内能完成、完成后可以展示出来的任务。”

## 6. 活动结束后的 CTA

**主 CTA：**

在活动结束后的 7 天内，选择并完成一个 Builder 小任务，然后把成果发布到社区或个人社交平台。

**推荐行动：**

- 选择一个任务方向：研究卡片、DApp 原型、AI 工作流、内容复盘、社区活动设计。
- 用一句话写清楚你的任务目标。
- 用 AI Agent 拆成 3-5 个执行步骤。
- 在 7 天内完成一个可展示版本。
- 发布成果，并附上一句话复盘：我用 AI 辅助完成了哪一步？

**示例 CTA 文案：**

“如果你是第一次参加 Builder 主题活动，不用急着做一个很大的项目。今晚先选一个 7 天任务：一张研究卡片、一个小原型、一篇复盘文章都可以。完成它，发布它，让自己真正进入 Builder 的行动状态。”
