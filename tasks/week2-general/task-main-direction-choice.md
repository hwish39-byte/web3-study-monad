# Week 2 主方向选择

我选择 **Dev** 作为主方向。

## 选择理由

Dev 方向最适合把前两周的学习转化为可验证的实际产出。Research 更偏信息整理，Ops 更偏流程和传播，而 Dev 能直接通过代码、合约交互、脚本和小工具证明自己真正理解 Monad 生态的技术栈。对 Web3 学习来说，能跑通链上交互、写出可复用 demo，比单纯阅读资料更容易形成作品集，也更方便后续找工作或参与生态项目。

## 服务的问题

Dev 方向主要服务的问题是：把 Monad 学习从“知道概念”推进到“能实际构建”。具体包括：

- 如何连接 Monad 测试网并完成基础链上交互
- 如何理解 EVM 兼容链上的开发流程
- 如何把合约、钱包、RPC、区块浏览器、前端交互串起来
- 如何沉淀成可展示、可复用、可继续扩展的小项目

## 本周最小产出

本周完成一个最小可运行 demo：

**Monad 测试网交互脚本 / 小工具**

最低目标：

- 配置 Monad 测试网 RPC
- 使用钱包私钥或浏览器钱包连接测试网
- 完成一次基础链上操作，例如查询余额、发送测试交易、读取合约数据
- 输出一份简短 README，记录环境配置、运行方式、遇到的问题和解决方法

如果时间允许，扩展为一个简单前端页面：

- 连接钱包
- 显示地址和余额
- 发起一笔测试交易或调用测试合约

## 参考资料

- Monad 官方文档
- Monad 测试网 RPC / 区块浏览器说明
- EVM / Solidity 基础资料
- ethers.js 或 viem 文档
- Hardhat / Foundry 基础教程
- Monad 官方 Discord、X、博客中的开发者资料

## Week 3 角色

Week 3 我承担 **Dev Builder / Technical Implementer** 角色。

主要职责是把学习内容落到代码和 demo 上，负责搭建开发环境、完成链上交互、记录技术路径，并把每周产出整理成可展示的项目资产。Research 和 Ops 的内容可以作为辅助输入，但我的核心贡献会集中在“能运行、能验证、能复盘”的技术产出上。
