# Space 小型策划案：《AI Agent 新手课：一个人如何搭起自己的 Builder 工作流》

## 1. 活动主题

**AI Agent 新手课：一个人如何搭起自己的 Builder 工作流**

本场 Space 面向 Web3、AI 与 Builder 方向的新手，围绕“如何用 AI Agent 提升个人构建能力”展开。活动不做复杂技术炫技，而是从新人真实场景出发，讲清楚 AI Agent 是什么、能帮 Builder 做什么，以及如何用它完成研究、选题、原型、内容和项目推进。

## 2. 活动目标

本场活动希望帮助新人完成三个转变：

- 从“听说 AI Agent 很火”转向“理解它能解决什么具体问题”。
- 从“想做 Builder 但不知道从哪里开始”转向“拥有一套可执行的个人工作流”。
- 从“只停留在学习概念”转向“能在 7 天内产出一个小作品或小成果”。

活动结束后，听众应能初步理解 AI Agent 与普通 AI 对话工具的区别，知道 Builder 常见工作流的组成，并能选择一个适合自己的轻量实践任务。

## 3. 目标听众

本场 Space 适合以下人群参与：

- 刚接触 Web3、AI 或 Builder 方向的新手。
- 想利用 AI 工具提高学习、研究、写作或开发效率的人。
- 有想法但缺少执行路径的个人 Builder。
- 非技术背景但希望参与 Web3 / AI 项目的人，如运营、内容、研究、社区、BD 等角色。
- 正在准备参与黑客松、开源项目、社区共创或个人作品集建设的人。

听众不需要具备高级编程基础，只要对 AI 工具、Web3 项目或个人创造有兴趣即可。

## 4. 活动希望解决的核心问题

本场活动聚焦解决一个核心问题：

**新人如何借助 AI Agent，把模糊的兴趣和想法转化成可执行、可展示、可持续迭代的 Builder 工作流？**

围绕这个核心问题，活动会进一步拆解几个具体问题：

- AI Agent 和普通 ChatGPT / AI 聊天工具有什么区别？
- 一个人做 Builder 时，哪些环节最适合交给 AI 辅助？
- 新人如何用 AI 完成项目研究、需求拆解、原型设计、代码辅助、内容发布和复盘？
- 非技术新人如何找到自己的参与位置？
- 如何设计一个 7 天内能完成的小型 Builder 任务，避免只学不做？

## 5. 活动为什么值得举办

AI Agent 正在降低个人 Builder 的启动门槛。过去，一个新人想从想法走到作品，往往需要同时具备研究、产品、设计、开发、运营等多种能力；现在，AI Agent 可以在资料整理、任务拆解、代码生成、内容表达和自动化执行等环节提供辅助，让个人更快完成从学习到产出的闭环。

同时，Web3 和 AI 领域的信息密度高、概念更新快，新人很容易陷入“看了很多，但不知道怎么开始”的状态。本场 Space 的价值在于把宏大的技术叙事拆成具体的个人行动路径：不是讨论 AI Agent 会不会改变世界，而是帮助每个参与者知道自己明天可以做什么。

这场活动也适合作为社区的新人入口。一方面，它能吸引技术、产品、运营、研究等不同背景的人参与；另一方面，它可以自然延伸出后续共学、作品打卡、黑客松组队、Demo 展示等活动，帮助社区从一次讨论沉淀为持续行动。

## 建议活动结构

- 0-5 分钟：开场与主题引入，说明为什么新人需要 Builder 工作流。
- 5-15 分钟：解释 AI Agent 的基本概念，以及它和普通 AI 对话工具的区别。
- 15-30 分钟：拆解个人 Builder 的核心工作流：研究、选题、拆任务、做原型、发布、复盘。
- 30-40 分钟：展示一个示例场景：如何用 AI Agent 从一个想法推进到一个小作品。
- 40-50 分钟：开放讨论：技术与非技术新人分别如何开始。
- 50-60 分钟：Q&A 与 7 天行动任务发布。

## 建议互动问题

- 你现在最想用 AI 帮你解决什么问题？
- 你更想成为技术型 Builder、内容型 Builder，还是研究/运营型 Builder？
- 如果给你 7 天时间做一个小作品，你会选择做什么？
- 你目前最大的阻碍是缺想法、缺技术、缺资源，还是缺持续行动？

## 7 天行动任务

活动结束后，邀请听众选择一个轻量任务完成：

- 用 AI Agent 整理一份 Web3 项目研究卡片。
- 做一个简单的 DApp / AI 工具原型。
- 搭建一个个人 Builder 主页或作品页。
- 写一篇“我如何用 AI 完成一次项目拆解”的复盘文章。
- 为一个开源项目或社区任务提交一次有效贡献。
