# 2026-07-10 学习日志

## 今日学习内容

### 1. Monad — 不止是"更快"

阅读 [Monad vs Ethereum](https://docs.monad.xyz/developer-essentials/differences)：

- Monad 是 EVM 兼容 L1，核心创新是 **并行执行（Parallel Execution）**
  - 以太坊：交易顺序执行，一个接一个，受限于单线程
  - Monad：交易并行执行，通过 Optimistic Execution 解决冲突，吞吐量大幅提升
- **延迟执行（Deferred Execution）**：共识层先确认交易顺序，执行层再并行处理，进一步提高 TPS
- **MonadBFT**：改进的共识机制，比 Tendermint / HotStuff 更快出块
- **EVM 兼容性**：现有的 Solidity 合约、工具链（Remix、Hardhat、Foundry、MetaMask）无需修改即可运行

### 2. 高 TPS 带来的产品可能

阅读 [10,000 TPS 会让什么成为可能](https://buildanything.so/zh/tracks/freshman)：

- **链上游戏**：每一步都在链上完成，不再依赖 off-chain 匹配 + on-chain 结算的混合方案
  - 全链棋牌（扑克、围棋、国际象棋）：每个落子 / 出牌都是一笔交易，低延迟 + 低费用才能玩得起来
- **高频 DeFi**：DEX 限价单、MEV 策略、高频套利机器人
- **社交 / 内容**：每一条点赞、评论、关注都是链上交易（Lens Protocol、Farcaster 模式）
- **链上预言机**：更频繁的价格更新，更精准的数据源

### 3. 高性能 dApp 最佳实践

阅读 [Best Practices](https://docs.monad.xyz/developer-essentials/best-practices)：

- **减少存储访问**：SLOAD 是最贵的操作之一，用 memory / calldata 替代 storage 操作
- **批量操作**：合并多次写入为一次，减少交易数量
- **事件（Event）优先**：日志比链上存储便宜，链下索引数据用 The Graph / 自行索引
- **异步设计**：合约只做核心逻辑，计算密集型工作放到链下

### 4. Gas 机制

阅读 [Gas Pricing](https://docs.monad.xyz/developer-essentials/gas-pricing)：

- Monad 的 Gas 模型与以太坊 EIP-1559 兼容
- Base Fee（基础费）→ 销毁 | Priority Fee（小费）→ 给验证者
- Monad 的高 TPS 意味着单位交易费用更低，适合高频交互场景
