# 2026-07-09 学习日志

## 今日学习内容

### 1. Web3 实习手册 — 智能合约开发

阅读 [智能合约开发章节](https://web3intern.xyz/zh/smart-contract-development/)：

- 智能合约 = 部署在链上的程序，不能被篡改，按代码逻辑自动执行
- Solidity 是以太坊上最主流的合约语言，语法接近 JavaScript / C++
- 核心数据类型：uint, address, bool, string, bytes, mapping, struct, array
- 关键关键字：public, private, internal, external, view, pure, payable, modifier
- 合约部署 = 将 Solidity 源码编译成 bytecode 并发送交易到链上 → 获得合约地址
- ABI（Application Binary Interface）：合约函数的接口描述，dApp 前端通过 ABI 调用合约

### 2. Solidity 基础

通过 [Solidity Docs](https://docs.soliditylang.org/) 学习：

- **状态变量**：存储在链上的数据，初始化后默认值（uint → 0, address → 0x0, bool → false）
- **函数可见性**：
  - `public` — 合约内外均可调用
  - `external` — 只能外部调用，内部不能用（比 public 省 gas）
  - `internal` — 仅合约内和子合约可调用
  - `private` — 仅合约内可调用
- **View / Pure**：不修改状态的函数，不用花 Gas（外部调用时）
- **Payable**：可以接收 ETH 的函数
- **Mapping**：键值对存储，`mapping(address => uint256)` 是最常用的数据结构
- **Msg.sender**：调用当前函数的地址，最重要的全局变量

### 3. Remix IDE 实操

使用 [Remix IDE](https://remix.ethereum.org/)：

- Remix 是浏览器端 Solidity IDE，免安装，适合快速原型
- 工作区：文件浏览器、Solidity 编译器、部署 & 交易面板
- 编译流程：选择编译器版本 → 编译 → 检查 ABI / Bytecode
- 部署流程：选择环境（Remix VM / Injected Provider / WalletConnect）→ Deploy
- 合约交互：部署后在 Deployed Contracts 面板调用函数

### 4. OpenZeppelin

了解 [OpenZeppelin](https://docs.openzeppelin.com/contracts/)：

- 行业标准的合约库，提供经过审计的 ERC20、ERC721、Ownable、AccessControl 等模块
- 用 OpenZeppelin 合约 = 站在巨人肩膀上，而不是从零写
- Ownable 模式：`onlyOwner` modifier 限制敏感函数仅合约部署者可调用

### 5. Monad Remix 部署指南

## 最小合约参考 — Counter.sol

```solidity
// SPDX-License-Identifier: MIT
pragma solidity ^0.8.20;

contract Counter {
    uint256 public count;

    function increment() public {
        count += 1;
    }

    function decrement() public {
        require(count > 0, "Counter: cannot go below zero");
        count -= 1;
    }

    function reset() public {
        count = 0;
    }
}
```

核心理解：
- `count` 是状态变量，存储在链上，每次 `increment()` 调用会改变链上状态 → 消耗 Gas
- `count` 是 `public`，自动生成 getter 函数，外部调用不消耗 Gas（因为是 view）
- `require` 用于条件检查，失败时 revert 交易并返还剩余 Gas

## 问题 / 卡点

- Remix 和本地 Foundry/Hardhat 环境的切换：Remix 适合快速原型，Foundry 适合正式开发（更快的编译、测试、调试）
- 合约升级问题：合约一旦部署不可篡改，需要代理模式（UUPS / Transparent Proxy）才能升级。短期内先做一次性部署，后续再学升级模式
- Gas 优化意识：在 Solidity 中，存储（SLOAD/SSTORE）是最贵的操作，后续要养成优化存储的习惯
